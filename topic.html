Introduction

HTTP

Http is talking to your friend in the middle of a party.

HTTP is a protocol for retrieving data such as HTML pages. It is the cornerstone of all data interchange on the Internet, and it is a client-server protocol, meaning that requests are started by the recipient, which is commonly the Web browser. Text, layout descriptions, photos, videos, scripts, and other sub-documents are fetched, and a full document is reconstructed from them.

HTTPs

Https is you and your friend talking alone in a soundproof room.

HTTPS is a secure version of HTTP, which is the most used protocol for sending data between a web browser and a website. This is especially crucial when users send sensitive information over the internet, such as bank account information. HTTPS should be used on any website. Websites that do not employ HTTPS are indicated differently. A green padlock in the URL bar in chrome indicates that the webpage is safe.Chrome and other browsers will flag any non-HTTPS websites as insecure.

SSL/TLS

The sound proof for HTTPs's private conversation.

SSL (Secure Sockets Layer) and TLS (Transport Layer Security), are protocols for creating authorised and encrypted connections. SSL protocol was deprecated with the introduction of TLS in 1999, these technologies are commonly referred to as "SSL" or "SSL/TLS."
